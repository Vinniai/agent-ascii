# agent-ascii — design context

## Design Context

### Users
Developers, AI-tool engineers, and operators who hand screenshots to agents and care about token cost. They read man pages, live in the terminal, and review diffs in plain text. They are evaluating whether `agent-ascii` is the right fit for CI drift gates, PR reviews, and context-cheap agent loops. They do not need to be persuaded with marketing — they need to see the tool work on realistic screenshots and copy a command.

### Brand Personality
Dry, precise, utilitarian. Think operator manual, not startup landing page. Three words: **mechanical**, **printed**, **exacting**. The tool's whole premise — faithfully re-rendering pixels as text — demands a site that treats text the same way: typeset, structured, not decorated.

### Aesthetic Direction
**Monochrome editorial, on paper.** Warm off-white background, ink-black text, one restrained accent (muted amber, evoking phosphor terminals without cosplaying them). Thin 1px rules replace cards for most content separation. Left-aligned everything; center alignment is reserved for the hero wordmark.

The dithering hero is configured to run in **ink-on-paper** — black halftone on cream — not cyan-on-black. This makes it feel like a printed specimen page rather than a trendy shader demo, and it mirrors what the CLI actually produces: dithered monochrome.

Type system:
- **Display / wordmark**: JetBrains Mono (we already brand with it — it literally renders our ASCII output). Used for headings, commands, labels.
- **Body**: Manrope. Humanist grotesque with slightly warm letterforms — pairs with the mono without feeling templated. Reject-list-safe.
- Fluid clamp() on headings, fixed rems on UI labels. ≥1.25 ratio between steps.

Color tokens (OKLCH):
- `--paper`    : oklch(0.98 0.008 80)   — warm off-white
- `--ink`      : oklch(0.18 0.01 80)    — softened near-black
- `--ink-muted`: oklch(0.50 0.01 80)    — secondary text
- `--rule`     : oklch(0.88 0.01 80)    — 1px rules, tinted toward ink
- `--accent`   : oklch(0.72 0.14 75)    — muted amber, used rarely

Anti-references:
- cyan-on-black shader demos
- gradient text
- icon-over-heading feature grids with identical cards
- glassmorphism / neon / purple-to-blue gradients
- generic tech-stack badge rows under a hero

### Design Principles
1. **The ASCII output is the hero.** Site chrome disappears so the renders can be the loud thing on the page.
2. **Left-aligned, ruled, printed.** Grids rest on thin 1px rules, not shadows or cards.
3. **One ink, one accent.** Black text on cream paper; amber appears only where action is required.
4. **Monospace for truth, sans for comfort.** Commands and labels live in JetBrains Mono; prose lives in Manrope.
5. **Pick strong sources.** Demonstrate on Apple/X mobile screenshots where layout and typography render richly — skip sparse pages that make the tool look weak.
